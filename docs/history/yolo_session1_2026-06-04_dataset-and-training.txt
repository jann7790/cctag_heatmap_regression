## The pipeline

The generator (`generate_cctag_dataset.py`) already emits YOLO-format labels in `labels_yolo/<name>.txt` (format `<cls> cx cy w h`, empty file = background negative). `prepare_yolo_dataset.py` just reshuffles those into the Ultralytics `images/{train,val}` + `labels/{train,val}` + `data.yaml` layout.

### 1. Generate the source dataset(s)

```bash
uv run python src/generate_cctag_dataset.py --num_images 2000 --seed 42 \
  --output_dir ./outputs/datasets/det_positive
```

Each source must contain `images/` and `labels_yolo/`. You can generate several (e.g. positives + hard negatives) and merge them in the next step.

### 2. Convert to the Ultralytics YOLO layout

```bash
uv run python src/prepare_yolo_dataset.py \
  --dataset_dir ./outputs/datasets/det_positive \
  --dataset_dir ./outputs/datasets/det_hard_negative \
  --output_dir ./outputs/datasets/yolo_detection \
  --train_ratio 0.9
```

Key points from the script:
- **Repeat `--dataset_dir`** to merge multiple source datasets; each source is prefixed (`s0_<name>__…`) so the `000000`-style numbering won't collide.
- **`--link_mode symlink`** (default) — places files as symlinks to save disk; use `--copy` if you need standalone/portable files (e.g. for a remote machine).
- **`--train_ratio`** (default 0.9), **`--seed`** (default 42) control the split.
- **`--class_name`** (default `cctag`) is the single class written to `data.yaml`.
- Empty label files are preserved as background negatives — intentional for a precision-leaning detector.
